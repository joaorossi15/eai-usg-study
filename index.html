O que é uma Ethical User Story?

Em engenharia de software, Histórias de Usuário são descrições curtas de requisitos que capturam o que um usuário precisa de um sistema. Elas geralmente seguem o formato: 

Como um [usuário],
Eu quero [capacidade],
para que [benefícios].

Esse formato ajuda equipes de desenvolvimento a transformar requisitos de alto nível em funcionalidades práticas para sistemas de software.

Ethical User Stories (EUS) estendem esse conceito para representar requisitos éticos de sistemas, especialmente em sistemas baseados em IA. Elas traduzem requisitos sob a ótica de princípios éticos, como transparência, privacidade, sustentabilidade, explicabilidade e beneficência, para que possam ser considerados durante o desenvolvimento do sistema.

Exemplo

Exemplo adaptado da literatura:

Como um passageiro usando o terminal de serviços,
Eu quero saber quais dados são coletados sobre mim e por quê,
para que eu possa me sentir seguro de que minhas informações estão protegidas.
Critérios de Aceitação 
- O sistema deve informar quais dados são coletados.
- O sistema deve informar a finalidade do uso desses dados.
- Essas informações devem estar disponíveis ao usuário de forma clara.
